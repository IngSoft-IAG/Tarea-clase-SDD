## What to build

Agregar 3–4 reservas de ejemplo en el `AppDbSeeder` (solo en entorno de desarrollo) para facilitar pruebas manuales. Las reservas deben cubrir distintas salas y rangos horarios (mañana/tarde) y una reserva adyacente para probar límites.

## Acceptance criteria

- [ ] Al ejecutar la API en modo desarrollo las reservas sample existen en la base InMemory.
- [ ] Las reservas permiten probar endpoints de disponibilidad manualmente.

## Blocked by

- None - can start immediately
